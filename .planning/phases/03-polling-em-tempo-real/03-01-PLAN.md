<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CitizenPortal → GET /api/upas/ | Endpoint público sem autenticação — resposta controlada pelo backend |
| CitizenPortal → google.com/maps | Link externo — dados de lat/lng injetados em URL |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | Botão Google Maps (`target="_blank"`) | mitigate | Usar `rel="noopener noreferrer"` em todo link externo — impede que a nova aba acesse `window.opener` (per D-4, locked decision) |
| T-03-02 | Tampering | UPAPublicaSerializer expondo lat/lng | accept | Coordenadas de UPAs públicas são informação pública; não há dados sensíveis expostos. UPASerializer admin já as expunha. |
| T-03-03 | Availability | Polling 10s por cliente ativo | accept | 10s é conservador; carga extra proporcional a usuários simultâneos. Para TCC (baixo volume), risco é irrelevante. Se necessário em produção, adicionar debounce ou backoff exponencial. |
| T-03-04 | Information Disclosure | lat/lng injetados em URL do Google Maps | accept | Dados públicos da UPA; URL não contém dados do usuário. Abrir em nova aba não vaza sessão do usuário. |
| T-03-05 | Denial of Service | Polling continua se aba fica em segundo plano | accept | Para TCC, impacto mínimo. A Page Visibility API (pausar polling quando oculto) é recomendada mas não bloqueante para este escopo. |
</threat_model>
